5. Important Service Functions

import_zerodha_portfolio(access_token) -> PortfolioSnapshot

Calls Zerodha holdings/positions APIs, validates response, maps external fields to internal holding schema, and labels unsupported instruments.

normalize_holdings(raw_holdings, instrument_master, sector_map) -> PortfolioSnapshot

Standardizes symbol, exchange, instrument token, asset type, quantity, price, market value, sector, and modeled flag.

load_price_matrix(symbols, start_date, end_date, source_policy) -> PriceMatrix

Loads aligned daily prices from cache/public source/Kite where available, then returns prices plus freshness and missing-data metadata.

calculate_returns(price_matrix, method="simple") -> ReturnMatrix

Calculates daily returns, aligns dates, and records dropped observations.

calculate_historical_var(portfolio_returns, value, confidence, horizon_days) -> RiskMetric

Finds empirical loss quantile, scales to selected horizon, and returns amount, percent, method, date range, and sample size.

calculate_expected_shortfall(portfolio_returns, value, confidence, horizon_days) -> RiskMetric

Averages tail losses beyond the VaR threshold and returns tail observation count.

calculate_component_var(weights, covariance_matrix, portfolio_value, confidence) -> list[Contribution]

Approximates marginal and component contribution to portfolio VaR for top risk driver explanations.

run_stress_scenario(portfolio, shock_vector, sector_map) -> StressResult

Applies configured shocks to symbols/sectors/factors and returns scenario loss plus contribution table.

analyze_spillover(portfolio_returns, sector_returns, commodity_returns, lags) -> SpilloverReport

Runs lagged correlation/regression scan, identifies strongest factor-lag pairs, and calculates commodity sensitivity score.

generate_copilot_context(portfolio_id, question) -> CopilotContext

Selects only necessary structured metrics and sanitized holdings context for LLM/MCP use.

6. Calculation Details

Calculation Input Implementation detail Caveat to display
Portfolio returns Return matrix and value weights r_p,t = sum(w_i * r_i,t) Assumes static current weights for historical simulation.
Historical VaR Portfolio returns, confidence, value Use empirical lower-tail percentile of returns, convert to positive loss amount. Past returns may not capture future shocks.
10-day VaR 1-day VaR Prototype may use square-root-time scaling. Scaling assumes independent returns; stress tests complement this.
Expected Shortfall Tail returns beyond VaR cutoff Average realized losses worse than VaR threshold. Can be unstable with small samples.
Commodity lag beta Portfolio/sector returns and lagged factor returns OLS regression or correlation scan across configured lags. Association is not proof of causality.

9. Copilot Prompt And Safety Contract

System policy for copilot:
- You are a portfolio risk explainer, not an investment adviser.
- Use only the structured risk context provided by tools.
- Do not recommend buying, selling, holding, or rebalancing a security.
- You may explain concentration, VaR, ES, stress loss, commodity sensitivity, and news risk.
- When data is partial or stale, say so clearly.
- If asked for trading advice, refuse briefly and provide risk diagnostics instead.
Prototype LLM mode
  • Provider is configurable.
  • Can be disabled in favor of deterministic memo.
  • Send summarized portfolio/risk context only.
Production LLM mode
  • Redact sensitive identifiers where possible.
  • Log tool calls and generated outputs for policy review.
  • Apply automated checks for recommendation language.

10. Error Codes And Handling

Code Meaning User message Developer action
AUTH_ZERODHA_FAILED Login/callback/session failed. Zerodha connection failed. Try again or load sample portfolio. Check callback URL, request token, state, and API credentials.
PORTFOLIO_EMPTY No holdings available. No supported holdings were found for risk analysis. Validate import parser and account contents.
PRICE_DATA_MISSING One or more symbols lack price history. Some holdings were excluded because price history was unavailable. Refresh cache or map symbols/instrument tokens.
RISK_SAMPLE_TOO_SMALL Not enough return observations. Risk estimates need more historical data to be reliable. Extend date range or fetch additional history.
NEWS_FEED_UNAVAILABLE RSS source failed. News context is currently unavailable; risk calculations are unaffected. Check feed URL, timeout, and parser.
LLM_UNAVAILABLE LLM provider failed or no key configured. AI explanation is unavailable; showing deterministic risk memo. Check provider key, quota, and request payload.

11. Security Implementation Notes

  • Never print access tokens, API secrets, request tokens, or LLM keys in logs.
  • Keep prototype credentials in local .env only.
  • Validate CSV file type, row count, numeric fields, and symbols before processing.
  • Use session-scoped portfolio ids; avoid exposing raw local file paths in API responses.
  • For production, use HTTPS, secure cookies, CSRF protection, encrypted token storage, and deletion workflow.
  • Add a visible disclaimer: "For educational risk analytics only. Not investment advice."
