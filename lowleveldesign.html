1. Two-Tier Data Architecture (BQuant + BLPAPI)

Key design principle BQuant (Python bql API) is the research environment; BLPAPI is the app data pipeline. They are complementary. Heavy cross-sectional research (Nifty 200 IC / Granger / DCC-GARCH / Diebold–Yilmaz / HMM / walk-forward) runs in BQuant and exports artifacts (JSON/Parquet) that the local BLPAPI-fed app consumes. Access note: BQuant API only — BQL is written in Python inside BQuant, not via the in-terminal/Excel query screen. NSE-Cogencis supplies Indian data + the news wire for the NLP layer.
Tier 1 — BQuant BQuant
  • Bloomberg-hosted Python (bql) environment — programmatic BQL, no terminal/Excel query screen needed.
  • Pulls the NIFTY 200 cross-section in a single query.
  • Fits DCC-GARCH / Diebold–Yilmaz spillover, HMM regimes, IC/Granger, walk-forward backtests.
  • Exports artifacts (regime history, signal rankings, spillover tables) for the local app.
Tier 2 — BLPAPI + Local App BLPAPI
  • Python BLPAPI session running against Bloomberg terminal on FRTL network.
  • Pulls adjusted daily prices, commodity futures, macro series, sector indices.
  • Caches all data locally in SQLite / Parquet under data/cache/bloomberg/.
  • Feeds risk engine, VaR backtester, spillover module, and copilot context.
  • BDH / BDP / BDS request patterns — documented in full below.

3. Bloomberg BLPAPI Integration

3.1 Session Setup

import blpapi

def create_bloomberg_session(host="localhost", port=8194) -> blpapi.Session:
    opts = blpapi.SessionOptions()
    opts.setServerHost(host)
    opts.setServerPort(port)
    session = blpapi.Session(opts)
    if not session.start():
        raise ConnectionError("Bloomberg session failed to start")
    if not session.openService("//blp/refdata"):
        raise ConnectionError("Could not open //blp/refdata service")
    return session

3.2 BDH — Historical Data Requests

def bdh(session, securities, fields, start_date, end_date,
        adj_split=True, adj_normal=True, periodicity="DAILY") -> pd.DataFrame:
    """Pull historical data for one or more securities."""
    svc = session.getService("//blp/refdata")
    req = svc.createRequest("HistoricalDataRequest")
    for sec in securities:
        req.getElement("securities").appendValue(sec)
    for fld in fields:
        req.getElement("fields").appendValue(fld)
    req.set("startDate", start_date)          # "YYYYMMDD"
    req.set("endDate", end_date)
    req.set("periodicitySelection", periodicity)
    req.set("adjustmentSplit", adj_split)
    req.set("adjustmentNormal", adj_normal)   # dividend adjustment
    session.sendRequest(req)
    # ... parse event loop, return pd.DataFrame indexed by date

3.3 BDP — Reference / Point-in-Time Data

def bdp(session, securities, fields) -> pd.DataFrame:
    """Pull reference data (non-time-series) for a list of securities."""

4. BQuant Research Notebooks

BQuant notebooks run in Bloomberg's hosted Python (bql) environment. They are the authoritative source for all spillover / factor outputs, exported as JSON/Parquet for the local BLPAPI-fed app.

4.1 BQL Query Patterns

# research Notebook — Factor IC Research
import bql
bq = bql.Service()

# Pull 3 years of adjusted daily returns for NIFTY 50 universe
univ = bq.univ.members('NIFTY Index')
ret_field = bq.data.px_last(
    dates=bq.func.range('-756d', '0d'),
    fill='prev'
).pct_change()

data = bq.execute(bql.Request(ret_field, univ))
returns_df = bql.combined_df(data).unstack('DATE').T

# Pull Brent crude returns (lagged factor)
brent_ret = bq.execute(bql.Request(
    bq.data.px_last(dates=bq.func.range('-756d', '0d'), fill='prev').pct_change(),
    ['CO1 Comdty']
))

# Pull USD/INR and India VIX as regime features
macro = bq.execute(bql.Request(
    bq.data.px_last(dates=bq.func.range('-756d', '0d'), fill='prev'),
    ['USDINR Curncy', 'INVIXN Index', 'GIND10YR Index']
))

4.2 Notebook Responsibilities and Exports

NotebookWhat it computesExported artifact
01_data_pull.ipynb Pull all required series via BQL; validate, align to Indian trading calendar; export to Parquet for use in other notebooks. equities_returns.parquet, commodity_returns.parquet, macro_series.parquet
02_factor_ic_research.ipynb For each commodity factor × lag combination, compute cross-sectional IC daily. Report IC mean, ICIR (IC/std), t-stat, and % positive IC days. factor_ic_results.json — ranked factor-lag table
03_granger_causality.ipynb Run Granger causality tests (lags 1–10) for each commodity factor vs. each sector index return. Report F-statistic and p-value. granger_results.json — commodity × sector × lag matrix
04_hmm_regime_detection.ipynb Fit 2-state Gaussian HMM on rolling vol, VIX, and return features. Label each date as regime 0 (low vol) or regime 1 (high vol). Walk-forward fitting to avoid lookahead. hmm_params.json, regime_history.parquet
05_walk_forward_backtest.ipynb For the top-ranked factor signals, run walk-forward backtest: train IC on rolling 252-day window, trade next 21 days, compute OOS Sharpe. walk_forward_results.json — OOS Sharpe, cumulative return series
06_cross_sectional_signals.ipynb Compute cross-sectional IC for each commodity factor across the full NIFTY 50 universe (not just the user's portfolio). Compare top-IC stocks to portfolio holdings. cross_sectional_ic.json
07_var_backtesting.ipynb Compute historical VaR violations using Bloomberg data. Run Kupiec LR test and Christoffersen independence test for both 95% and 99% confidence levels. var_backtest_results.json

6. Key Service Function Contracts

6.1 Bloomberg Data Layer

BloombergCache.get_prices(ticker, start, end) -> pd.DataFrame

Cache-first. Returns Parquet-cached series if available and fresh; otherwise calls BLPAPI bdh(), writes to cache, and returns. Raises DataUnavailableError if neither cache nor BLPAPI succeeds.

TickerRegistry.resolve(zerodha_symbol) -> str

Maps Zerodha trading symbol (e.g. RELIANCE) to Bloomberg ticker (e.g. RELIANCE IN Equity). Uses bloomberg_tickers.json; falls back to {symbol} IN Equity pattern with validation.

build_return_matrix(symbols, start, end, source="bloomberg") -> ReturnMatrix

Loads adjusted close prices for all symbols from Bloomberg cache, aligns to common trading dates (Indian calendar), computes simple daily returns, reports dropped dates and missing symbols.

ResearchArtifactLoader.load(artifact_name) -> dict | pd.DataFrame

Loads a named artifact from data/cache/research_artifacts/. Validates schema on load. Raises ArtifactNotFoundError with clear message if artifact hasn't been exported from research yet.

6.2 Risk Engine

calculate_historical_var(portfolio_returns, value, confidence, horizon_days) -> VaRResult

Empirical lower-tail quantile of portfolio return distribution. Scales to horizon using square-root-of-time. Returns amount, percent, sample size, and date range.

calculate_parametric_var(weights, covariance_matrix, value, confidence, horizon_days) -> VaRResult

Delta-normal VaR using current weights and Bloomberg-derived covariance matrix. Assumes multivariate normal returns — caveat reported in output.

calculate_monte_carlo_var(weights, covariance_matrix, value, confidence, horizon_days, n_sim=10000) -> VaRResult

Cholesky decomposition of covariance matrix; simulate correlated returns; compute portfolio P&L distribution empirical quantile.

calculate_expected_shortfall(portfolio_returns, value, confidence, horizon_days) -> ESResult

Mean of returns worse than VaR threshold. Returns tail observation count. Flags instability if tail obs < 30.

calculate_component_var(weights, covariance_matrix, portfolio_value, confidence) -> list[Contribution]

Component VaR = weight × (covariance of holding with portfolio) / portfolio_vol × z-score × sqrt(horizon) × portfolio_value. Marginal VaR = component_var / weight.

run_stress_scenario(portfolio, shock_vector, sector_map) -> StressResult

Applies shocks to holdings by symbol or sector. Built-in scenarios: covid_march2020 (equity −35%), rate_hike_100bps (rate-sensitive −12%), crude_shock_2022 (oil sector −18%, input-cost sectors −8%), inr_depreciation_10pct (IT +6%, import-heavy −9%), election_2024_vol (broad −5% with +30% vol).

6.3 VaR Backtesting

run_kupiec_test(violations, n_obs, confidence) -> KupiecResult

Likelihood ratio test: LR = -2 * ln[(1-p)^(n-v) * p^v] + 2 * ln[(1-v/n)^(n-v) * (v/n)^v]. Chi-squared distributed with 1 df. p-value from chi2 CDF. Pass if p-value > 0.05.

run_christoffersen_test(violation_series) -> ChristoffersenResult

Independence test on violation sequence. Estimates transition probabilities between violation/no-violation states. Tests whether violations cluster (independence hypothesis). LR statistic, p-value, interpretation.

6.4 Spillover & Factor Research

fit_dcc_garch(returns_matrix) -> DCCResult

Univariate GARCH(1,1) per series, then a DCC model; returns time-varying conditional correlations between each sector index and the commodity/macro factors. Surfaces crisis correlation spikes (COVID-2020, 2022) — the headline spillover result.

diebold_yilmaz_connectedness(returns_matrix, var_lag, fevd_horizon) -> SpilloverTable

Fits a VAR, computes the generalized forecast-error variance decomposition, and returns total / directional (to, from) / net / pairwise connectedness across sectors + commodities + macro. Identifies who transmits and who receives risk.

calculate_ic(factor_returns_lagged, forward_stock_returns) -> ICResult

Rank correlation (Spearman) between lagged factor return and forward stock return cross-section. Returns IC, IC standard deviation, ICIR = IC_mean / IC_std, t-stat = ICIR * sqrt(n_periods), p-value.

run_granger_test(x_series, y_series, max_lag) -> GrangerResult

Uses statsmodels.tsa.stattools.grangercausalitytests. Returns F-statistic and p-value per lag. Reports significant lags at 5% threshold.

calculate_portfolio_commodity_sensitivity(portfolio_returns, factor_returns_dict, lags) -> SpilloverReport

OLS regression of portfolio returns on lagged commodity factor returns. Reports beta, t-stat, R-squared, and confidence indicator per factor-lag pair. Flags "not significant" explicitly.

6.5 ML Layer (Loads research Artifacts)

RegimeDetector.current_regime(date) -> RegimeLabel

Loads regime_history.parquet artifact. Returns regime label (0=low_vol, 1=high_vol), posterior probabilities, and consecutive days in current regime for the requested date.

RegimeConditionedVaR.compute(portfolio_returns, regime_history, value, confidence) -> RegimeVaRResult

Splits return history by regime label. Computes historical VaR separately for each regime. Returns both VaR estimates plus sample sizes. Current regime VaR highlighted as the active risk estimate.

6.6 Copilot Layer

generate_copilot_context(portfolio_id, question, allowed_context) -> CopilotContext

Selects minimal structured data needed to answer the question. Pulls from: risk summary, VaR backtest result, signal ranking, regime status, news tags. Redacts raw price data. Returns structured JSON passed to LLM/MCP.

generate_deterministic_memo(portfolio_id, style) -> str

Builds a complete risk memo from structured outputs without calling an LLM. Sections: portfolio summary, VaR comparison (3 methods), backtest verdict, current regime, stress scenarios, top risk drivers, commodity exposure, news flags, methodology caveats. Safe to use when LLM is unavailable.

10. Copilot System Policy

System policy:
- You are Drishti's portfolio risk explainer, not an investment adviser.
- Use ONLY structured outputs from Risk MCP tools. Never invent numbers.
- Do not recommend buying, selling, holding, or rebalancing any security.
- You may explain: VaR (and compare methods), ES, backtest results, regime
  implications, component contributions, stress losses, IC/t-stat signal
  quality, Granger causality findings, and news risk flags.
- When the backtest fails (Christoffersen), explain what clustering of
  violations means — do not say the portfolio is safe or unsafe.
- When data is partial, stale, or an artifact is missing, say so explicitly.
- If asked for trade advice, refuse with one sentence and redirect to the
  relevant risk diagnostic instead.

13. Security Implementation Notes

  • Never log Bloomberg access tokens, Zerodha access tokens, LLM API keys, or raw price data to console or files.
  • All secrets in local .env — never committed to version control.
  • Bloomberg raw price files gitignored — cite data source in code comments and reports only.
  • Validate CSV file type, row count, numeric fields, and symbol format before processing.
  • Session-scoped portfolio IDs — never expose local file paths in API responses.
  • research notebooks should not contain hardcoded credentials — use Bloomberg session authentication natively.
  • Add visible disclaimer in UI and all reports: "For educational risk analytics only. Not investment advice."
