How to Use Drishti

Option 1 — Sample Portfolio (Instant)

  1. Click Load Sample Portfolio on the Overview tab.
  2. Click Run Risk Analysis. Results appear in ~3 seconds.
  3. Switch to Risk Detail, Factor Research, and Spillover tabs for deeper analytics.
  4. Go to the Copilot tab and click Generate Memo for a plain-English risk summary.

Option 2 — Upload Your Own Portfolio (CSV)

  1. Download your holdings CSV from your broker (see Broker Guides below).
  2. Click Upload CSV on the Overview tab and select your file.
  3. Click Run Risk Analysis.

The CSV must have columns: symbol, quantity, avg_price. NSE symbols only (e.g. RELIANCE, not RELIANCE.NS).

Broker CSV Download Guides

Zerodha (Kite)
  1. Log in to kite.zerodha.com
  2. Go to Portfolio → Holdings
  3. Click the download icon (↓) at the top right → select CSV
  4. Rename columns: Symbol → symbol, Qty. → quantity, Avg. cost → avg_price

  3. Map: Stock Symbol → symbol, Quantity → quantity, Average Price → avg_price

  2. Export icon → Download CSV
  3. Map: Instrument → symbol, Quantity → quantity, Average Price → avg_price
AngelOne

  2. Download → CSV/Excel
  3. Map: NSE Code → symbol, Quantity → quantity, Avg Price → avg_price
5Paisa
  1. Log in to 5paisa.com → Holdings
  2. Export → Download as CSV
  3. Map: Script Name → symbol, Qty → quantity, Avg Rate → avg_price

Zerodha MCP Server Setup

The kite-mcp server lets AI copilots read live holdings via Kite Connect API — no manual CSV export needed.

Prerequisites

  • Zerodha account with Kite Connect API access (₹2000/month)
  • API key and secret from developers.kite.trade
  • Node.js 18+ installed

Setup

  1. Clone: git clone https://github.com/zerodha/kite-mcp
  2. Install: cd kite-mcp && npm install
  3. Set env vars: KITE_API_KEY, KITE_API_SECRET
  4. Start: npm start (port 3000)
  5. Authenticate via the Kite login URL provided by the server
kite-mcp requires a paid Kite Connect subscription. For course demo purposes, CSV upload is sufficient.

Methodology

Value at Risk (VaR)

VaR at confidence level $\alpha$ is the loss threshold not exceeded with probability $\alpha$:

$$\text{VaR}_\alpha = -\text{Quantile}_{1-\alpha}(r_1, \ldots, r_T)$$
  • Historical (non-overlapping): Empirical quantile of 10-day block returns. No distribution assumption.
  • Parametric (delta-normal): $\text{VaR} = \sigma_p \cdot z_\alpha$, $\sigma_p = \sqrt{w^\top \Sigma w}$. $\sqrt{10}$ horizon scaling.
  • GARCH-FHS: GARCH(1,1) standardises residuals; bootstrap preserves empirical tail shape.

Expected Shortfall

$$\text{ES}_\alpha = -\frac{1}{(1-\alpha) T} \sum_{t:\, r_t < -\text{VaR}_\alpha} r_t$$

ES is coherent (subadditive); used in Basel IV at 97.5% confidence.

VaR Backtesting

Kupiec (1995): LR test on violation frequency.

$$LR_{POF} = -2\ln\frac{p^x(1-p)^{T-x}}{\hat{p}^x(1-\hat{p})^{T-x}} \sim \chi^2_1$$

Christoffersen (1998): Tests whether violations cluster (independence test).

HMM Volatility Regimes

$$r_t \mid s_t = k \sim \mathcal{N}(\mu_k, \sigma_k^2), \quad P(s_t \mid s_{t-1}) = A$$

2-state Gaussian HMM. Walk-forward refit every 21 days. Canonical labeling by rolling-vol emission mean prevents label-switching.

DCC-GARCH Dynamic Correlations

$$Q_t = (1-a-b)\bar{Q} + a\,\epsilon_{t-1}\epsilon_{t-1}^\top + b\,Q_{t-1}$$
$$R_t = \text{diag}(Q_t)^{-1/2}\,Q_t\,\text{diag}(Q_t)^{-1/2}$$

Engle (2002) 2-step estimator. $R_t$ is the time-varying correlation matrix.

Diebold-Yilmaz Connectedness

$$\theta_{ij}^g(H) = \frac{\sigma_{jj}^{-1}\sum_{h=0}^{H-1}(e_i^\top A_h \Sigma e_j)^2}{\sum_{h=0}^{H-1}e_i^\top A_h \Sigma A_h^\top e_i}$$

Pesaran-Shin GFEVD — order-invariant. VAR lag by AIC, capped at 5.

Time-Series IC

$$IC_t(\text{lag}) = \text{Corr}(r_{\text{factor},\, t-\text{lag}},\, r_{\text{sector},\, t}) \quad \text{63-day window}$$

ICIR = mean IC / std IC. Benjamini-Hochberg FDR correction at α=0.05 across ~200+ tests.

Glossary

BH FDR (Benjamini-Hochberg)
Controls the False Discovery Rate across multiple tests. Applied to all IC p-values at α=0.05.
Component VaR
Stock $i$'s contribution to portfolio VaR: $w_i \cdot (\partial \text{VaR}/\partial w_i)$. Components sum to total VaR.
DCC-GARCH
Dynamic Conditional Correlation GARCH (Engle 2002). Time-varying correlation matrix from GARCH-standardised residuals.
Diebold-Yilmaz Index
Cross-market shock transmission via Pesaran-Shin GFEVD. Total connectedness = % forecast-error variance from cross-market shocks.
ES (Expected Shortfall)
Average loss when VaR is breached. Coherent risk measure. Basel IV: 97.5% ES.
GARCH-FHS
Filtered Historical Simulation. GARCH standardises residuals; bootstrap preserves empirical fat tails.
Granger Causality
Tests whether lagged $X$ improves prediction of $Y$ beyond $Y$'s own lags. Predictive, not structural causality.
HMM (Hidden Markov Model)
Unobserved discrete state (low-vol/high-vol) generates observed returns. Baum-Welch EM inference.
IC (Information Coefficient)
Rolling Pearson correlation between lagged factor and sector return. ICIR = IC mean / IC std.
Kupiec Test
Proportion-of-Failures LR test for VaR. $H_0$: violation rate = $1-\alpha$. $\chi^2_1$ distribution.
VaR (Value at Risk)
Max loss not exceeded with probability $\alpha$ over horizon $h$. Not a worst-case — just a quantile.
