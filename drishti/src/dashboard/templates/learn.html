How to Use Drishti

Option 1 — Sample Portfolio (Instant)

  1. Click Load Sample Portfolio on the Overview tab.
  2. Click Run Risk Analysis. Results appear in ~3 seconds.
  3. Switch to Risk Detail, Factor Research, and Spillover tabs for deeper analytics.
  4. Go to the Copilot tab and click Generate Memo for a plain-English risk summary.

Option 2 — Upload Your Own Portfolio (CSV)

  1. Download your holdings CSV from your broker (see Broker Guides below).
  2. Click Upload CSV on the Overview tab and select your file.
  3. Click Run Risk Analysis.

The CSV must have columns: symbol, quantity, avg_price. NSE symbols only (e.g. RELIANCE, not RELIANCE.NS).

Zerodha MCP Server Setup

The Zerodha MCP server (kite-mcp) lets AI copilots read your live holdings directly via the Kite Connect API — no manual CSV export needed.

Prerequisites

  • Zerodha account with Kite Connect API access (₹2000/month subscription)
  • API key and secret from developers.kite.trade
  • Node.js 18+ installed

Setup Steps

  1. Clone the kite-mcp server: git clone https://github.com/zerodha/kite-mcp
  2. Install dependencies: cd kite-mcp && npm install
  3. Set environment variables: KITE_API_KEY, KITE_API_SECRET
  4. Start the server: npm start — runs on port 3000 by default
  5. In your MCP client config, add the Kite MCP server endpoint
  6. Authenticate via the Kite login URL provided by the server
kite-mcp gives live portfolio data but requires a paid Kite Connect API subscription. For course demo purposes, the CSV upload method is sufficient.

Methodology

Value at Risk (VaR)

VaR at confidence level $\alpha$ is the loss threshold not exceeded with probability $\alpha$. For a portfolio with P&L series $r_t$:

$$\text{VaR}_\alpha = -\text{Quantile}_{1-\alpha}(r_1, \ldots, r_T)$$

Three methods implemented:

  • Historical (non-overlapping windows): Empirical quantile of 10-day non-overlapping block returns. No distribution assumption. Uses $\lfloor T/10 \rfloor$ observations.
  • Parametric (delta-normal): Assumes multivariate normal; VaR = $\sigma_p \cdot z_\alpha$ where $\sigma_p = \sqrt{w^\top \Sigma w}$. 10-day horizon via $\sqrt{10}$ scaling (stated assumption).
  • GARCH-FHS: GARCH(1,1) filters residuals; bootstrap from standardized residuals using GARCH-forecasted vol. Preserves empirical tail shape without normal assumption.

Expected Shortfall (CVaR)

Expected loss given that VaR is breached — the average of the worst $(1-\alpha)$ fraction of outcomes:

$$\text{ES}_\alpha = -\frac{1}{(1-\alpha) T} \sum_{t:\, r_t < -\text{VaR}_\alpha} r_t$$

ES is coherent (satisfies subadditivity) whereas VaR is not. ES is used in the Basel III/IV standardised approach for market risk.

VaR Backtesting

Kupiec (1995) Proportion-of-Failures test: Tests whether the observed violation rate equals the theoretical rate $p = 1 - \alpha$. Likelihood ratio:

$$LR_{POF} = -2\ln\frac{p^x(1-p)^{T-x}}{\hat{p}^x(1-\hat{p})^{T-x}} \sim \chi^2_1$$

Christoffersen (1998) Independence test: Tests whether violations cluster. A correct VaR model produces i.i.d. Bernoulli hits; clustering implies the model underestimates tail dependence.

HMM Volatility Regimes

A 2-state Gaussian Hidden Markov Model fits the portfolio return series. States are canonically labeled by emission mean of rolling realised volatility: state 0 = low-vol, state 1 = high-vol.

$$r_t \mid s_t = k \sim \mathcal{N}(\mu_k, \sigma_k^2), \quad P(s_t \mid s_{t-1}) = A$$

Walk-forward refitting every 21 trading days prevents label-switching. Regime-conditioned VaR conditions the return quantile on the current state posterior.

DCC-GARCH Dynamic Correlations

Engle (2002) 2-step estimator. Step 1: fit univariate GARCH(1,1) to each series, extract standardised residuals $\epsilon_t$. Step 2: fit DCC to the $\epsilon_t$ matrix:

$$Q_t = (1-a-b)\bar{Q} + a\,\epsilon_{t-1}\epsilon_{t-1}^\top + b\,Q_{t-1}$$
$$R_t = \text{diag}(Q_t)^{-1/2}\,Q_t\,\text{diag}(Q_t)^{-1/2}$$

$R_t$ is the time-varying correlation matrix. The chart shows $R_t[i,j]$ for the most informative factor–sector pairs.

Diebold-Yilmaz Connectedness

Pesaran-Shin generalised FEVD (order-invariant). VAR($p$) fitted to the commodity/macro + sector return panel; lag $p$ selected by AIC, capped at 5:

$$\theta_{ij}^g(H) = \frac{\sigma_{jj}^{-1}\sum_{h=0}^{H-1}(e_i^\top A_h \Sigma e_j)^2}{\sum_{h=0}^{H-1}e_i^\top A_h \Sigma A_h^\top e_i}$$

Total connectedness = share of forecast-error variance attributable to cross-market shocks.

Time-Series IC (Information Coefficient)

A commodity return at time $t$ is scalar — identical for all stocks — so cross-sectional IC is undefined. Rolling Pearson correlation between lagged factor return and sector return:

$$IC_t(\text{lag}) = \text{Corr}(r_{\text{factor},\, t-\text{lag}},\, r_{\text{sector},\, t}) \quad \text{over a 63-day window}$$

ICIR = mean IC / std IC. Granger causality tests predictive power beyond autoregressive terms. Benjamini-Hochberg FDR correction at α = 0.05 applied to all p-values.

Glossary

BH FDR (Benjamini-Hochberg)
Multiple-testing correction that controls the False Discovery Rate — the expected fraction of rejected nulls that are false positives. Applied to all IC p-values at α = 0.05.
Component VaR
Contribution of stock $i$ to portfolio VaR: $\text{CVaR}_i = w_i \cdot (\partial \text{VaR}/\partial w_i)$. Components sum to total portfolio VaR.
DCC-GARCH
Dynamic Conditional Correlation GARCH. Models time-varying correlations between return series. Engle (2002). Captures changing co-movement during stress periods.
Diebold-Yilmaz (DY) Index
Measures cross-market shock transmission via Pesaran-Shin generalised FEVD. Total connectedness = % of variance from cross-market shocks. High → tight co-movement, less diversification benefit.
ES (Expected Shortfall / CVaR)
Average loss in the worst $(1-\alpha)$ fraction of scenarios. ES is coherent (subadditive); Basel IV uses 97.5% ES as the primary market risk measure.
GARCH-FHS
Filtered Historical Simulation. GARCH(1,1) standardises residuals; bootstrap from them using GARCH-forecast vol. Captures fat tails without full parametric assumption.
Granger Causality
Tests whether lagged values of $X$ improve prediction of $Y$ beyond $Y$'s own lags. Predictive causality, not structural.
HMM (Hidden Markov Model)
Probabilistic model where an unobserved discrete state sequence (here: low-vol / high-vol regime) generates observed returns. State inference via Baum-Welch EM; prediction via Viterbi.
IC (Information Coefficient)
Rolling Pearson correlation between a lagged factor return and a sector return. Measures predictive signal strength. ICIR = IC mean / IC std = signal-to-noise ratio.
Kupiec Test
Proportion-of-Failures (POF) likelihood-ratio test for VaR. Tests $H_0$: observed violation rate = $(1-\alpha)$. $\chi^2_1$ under $H_0$. Does not test independence of violations.
VaR (Value at Risk)
Maximum portfolio loss not exceeded with probability $\alpha$ over horizon $h$. At 99% 10-day: 1% chance the 10-day loss exceeds VaR. Not a worst-case — just a quantile.
