Drishti — Complete Methodology

Every mathematical technique, equation, derivation, design decision, and implementation detail behind the Drishti quantitative risk platform. Written for academic and examination purposes.

1. Portfolio Construction & Return Computation

Foundation

1.1 What is a portfolio return?

A portfolio is a collection of $n$ assets held in weights $\mathbf{w} = (w_1, \ldots, w_n)^\top$ where $\sum_i w_i = 1$. At each time $t$, the portfolio's simple (arithmetic) return is the weighted average of individual asset returns:

Portfolio return
$$R_{p,t} = \sum_{i=1}^{n} w_i R_{i,t} = \mathbf{w}^\top \mathbf{R}_t$$

Individual asset returns are computed from adjusted close prices (Bloomberg PX_LAST with adj_split=True, adj_normal=True):

$$R_{i,t} = \frac{P_{i,t} - P_{i,t-1}}{P_{i,t-1}} = \frac{P_{i,t}}{P_{i,t-1}} - 1$$

We use simple returns (not log returns) because portfolio aggregation $R_p = \mathbf{w}^\top \mathbf{R}$ is exact for simple returns but only approximate for log returns. Log returns are used only where compounding is required (weekly aggregation for Granger tests).

1.2 Covariance matrix estimation

The sample covariance matrix is estimated on the aligned return matrix $\mathbf{R} \in \mathbb{R}^{T \times n}$:

$$\hat{\boldsymbol{\Sigma}} = \frac{1}{T-1} \sum_{t=1}^{T} (\mathbf{R}_t - \bar{\mathbf{R}})(\mathbf{R}_t - \bar{\mathbf{R}})^\top$$

The portfolio variance follows as the scalar quadratic form $\sigma_p^2 = \mathbf{w}^\top \hat{\boldsymbol{\Sigma}} \mathbf{w}$.

2. IPO Truncation Problem & Min-History Filter

Data Engineering

2.1 The problem

When building a return matrix from multiple assets, a naive dropna() after alignment truncates the entire matrix to the shortest series. If one stock listed in 2023, the full 2018–2026 history of all other stocks is discarded. This is a classic survivorship/truncation bug that artificially reduces effective sample size.

Example: 49 stocks with history since 2018 + 1 IPO from 2024 → dropna() produces a 2-year matrix. Historical VaR, GARCH, and backtests all break.

2.2 The fix: per-column minimum history filter

Before alignment, drop any column whose non-NaN observation count is below a configurable threshold:

Filter criterion
$$\text{Keep ticker } i \iff |\{t : P_{i,t} \neq \text{NaN}\}| \geq D_{\min}$$

Drishti uses $D_{\min} = 756$ trading days (≈ 3 years, configurable via DRISHTI_MIN_HISTORY_DAYS). This preserves the full long-run history of established names while excluding very recent IPOs.

After filtering, ffill() fills at most a few missing days (holidays, halts), then the remaining NaN rows at the front of each series are handled series-by-series rather than by a global dropna().

3. Historical Simulation VaR

Market Risk

3.1 Definition and motivation

Value at Risk at confidence level $\alpha$ and horizon $h$ is the loss threshold not exceeded with probability $\alpha$. Formally, if $L = -R_p$ is the loss:

VaR definition
$$\text{VaR}_\alpha^h = \inf\{l \in \mathbb{R} : \Pr(L_h > l) \leq 1-\alpha\} = -F_{R}^{-1}(1-\alpha)$$

Historical simulation makes no distributional assumption — it directly uses the empirical distribution of past returns as a proxy for the future. The 1-day $99\%$ VaR is simply the 1st percentile of the empirical return distribution:

$$\widehat{\text{VaR}}_{0.99}^{1} = -\hat{F}^{-1}(0.01) = -\text{Quantile}(\{R_{p,t}\}_{t=1}^{T},\; 0.01)$$

3.2 Multi-day horizon: non-overlapping windows

For a $h$-day VaR, the correct approach aggregates returns over non-overlapping windows of length $h$:

h-day non-overlapping return
$$R_p^{(h,k)} = \sum_{j=0}^{h-1} R_{p, kh+j}, \qquad k = 0, 1, \ldots, \left\lfloor T/h \right\rfloor - 1$$

The empirical quantile is then taken over $\{R_p^{(h,k)}\}$. This preserves any serial dependence (volatility clustering) in the actual return process.

Why not $\sqrt{h}$ scaling? The $\sqrt{h}$ rule assumes i.i.d. returns (a random walk). Financial returns exhibit volatility clustering — large moves cluster together. Using $\sqrt{h}$ scaling when returns are not i.i.d. understates multi-day VaR during turbulent periods. The non-overlapping approach is an empirical estimate that automatically captures clustering effects.

Fallback: if $\lfloor T/h \rfloor < 30$ windows are available, Drishti falls back to $\sqrt{h}$ scaling with an explicit warning in the output.

4. Parametric (Delta-Normal) VaR

Market Risk

4.1 The multivariate-normal assumption

Under the assumption $\mathbf{R}_t \sim \mathcal{N}(\boldsymbol{\mu}, \boldsymbol{\Sigma})$, the portfolio return is univariate normal:

$$R_{p,t} = \mathbf{w}^\top \mathbf{R}_t \sim \mathcal{N}\!\left(\mathbf{w}^\top\boldsymbol{\mu},\; \mathbf{w}^\top\boldsymbol{\Sigma}\mathbf{w}\right)$$

For risk purposes the mean term is small relative to the standard deviation at short horizons (the "zero-mean" approximation), so:

Parametric 1-day VaR (closed form)
$$\text{VaR}_\alpha^1 = -z_{1-\alpha} \cdot \sigma_p = |z_{1-\alpha}| \cdot \sqrt{\mathbf{w}^\top\boldsymbol{\Sigma}\mathbf{w}}$$

where $z_{1-\alpha} = \Phi^{-1}(1-\alpha)$ is the standard-normal quantile. At 99% confidence, $z_{0.01} = \Phi^{-1}(0.01) \approx -2.326$.

4.2 Horizon scaling

Under i.i.d. normality, $h$-day returns are normally distributed with variance $h\sigma_p^2$, giving:

Parametric h-day VaR
$$\text{VaR}_\alpha^h = |z_{1-\alpha}| \cdot \sigma_p \cdot \sqrt{h}$$

The $\sqrt{h}$ rule is stated as an explicit assumption in Drishti's output, not claimed as empirically valid. It is provided for comparison with the non-overlapping historical estimate.

4.3 Verification: closed-form test

For a single asset with $\sigma = 2\%$/day and portfolio value $V = 1{,}000{,}000$:

$$\text{VaR}_{0.99}^1 = 2.326 \times 0.02 \times 1{,}000{,}000 = 46{,}520$$

This exact value is verified in tests/test_es_contribution.py::test_parametric_var_closed_form.

5. GARCH-FHS VaR

Market Risk

5.1 Why GARCH-FHS is genuinely different

Parametric VaR assumes a constant volatility. In reality, equity volatilities cluster — large moves tend to follow large moves (Mandelbrot 1963, Engle 1982). GARCH-FHS (Filtered Historical Simulation) combines GARCH's time-varying volatility forecast with the empirical tail shape of standardized residuals. It is not the same as parametric VaR with a GARCH-estimated $\sigma$.

5.2 GARCH(1,1) — the volatility model

Let $r_t$ be the portfolio return at time $t$. The GARCH(1,1) model decomposes returns into a conditional mean (assumed zero) and a volatility-scaled shock:

GARCH(1,1) specification
$$r_t = \sigma_t \varepsilon_t, \qquad \varepsilon_t \stackrel{\text{i.i.d.}}{\sim} \mathcal{N}(0,1)$$ $$\sigma_t^2 = \omega + \alpha r_{t-1}^2 + \beta \sigma_{t-1}^2$$

with parameters $\omega > 0$, $\alpha \geq 0$, $\beta \geq 0$, $\alpha + \beta < 1$ (stationarity). The unconditional (long-run) variance is:

$$\bar{\sigma}^2 = \frac{\omega}{1 - \alpha - \beta}$$

Interpretation: $\alpha$ measures how quickly variance responds to new shocks (ARCH effect); $\beta$ measures persistence (how long elevated volatility lasts). High $\beta$ (e.g. 0.88 in typical equity data) means volatility is very persistent.

5.3 GARCH-FHS procedure

Fit GARCH(1,1) to the full history of portfolio returns, obtaining parameters $\hat\omega, \hat\alpha, \hat\beta$ and the conditional volatility path $\hat\sigma_1, \ldots, \hat\sigma_T$.
Extract standardized residuals: $$\hat{z}_t = \frac{r_t}{\hat\sigma_t}, \quad t=1,\ldots,T$$ These are approximately i.i.d. with zero mean and unit variance, but preserving the empirical tail shape (fat tails, skewness).
Forecast next-period volatility: $$\hat\sigma_{T+1}^2 = \hat\omega + \hat\alpha r_T^2 + \hat\beta \hat\sigma_T^2$$
Bootstrap simulated returns by sampling $B=10{,}000$ values from $\{\hat z_t\}$ with replacement, then scaling: $$\tilde r^{(b)} = \hat z^{(b)} \cdot \hat\sigma_{T+1}, \quad b=1,\ldots,B$$
Read off VaR as the empirical quantile of the bootstrap distribution: $$\widehat{\text{VaR}}_{0.99}^{\text{FHS}} = -\text{Quantile}\!\left(\{\tilde r^{(b)}\}, 0.01\right)$$

5.4 Multi-day FHS

For $h > 1$ day, Drishti simulates full GARCH paths rather than scaling by $\sqrt{h}$. Each simulated path recursively propagates GARCH dynamics:

$$\tilde r_{T+j}^{(b)} = \hat z_j^{(b)} \cdot \tilde\sigma_{T+j}, \quad \tilde\sigma_{T+j}^2 = \hat\omega + \hat\alpha (\tilde r_{T+j-1}^{(b)})^2 + \hat\beta \tilde\sigma_{T+j-1}^2$$ $$R_h^{(b)} = \sum_{j=1}^{h} \tilde r_{T+j}^{(b)}$$

The 10-day GARCH-FHS VaR is always strictly greater than the 1-day VaR, as verified in tests.

6. Expected Shortfall (CVaR)

Market Risk

6.1 Definition

Expected Shortfall (ES), also called Conditional VaR (CVaR) or Expected Tail Loss, is the expected loss given that the loss exceeds VaR:

ES definition
$$\text{ES}_\alpha = \mathbb{E}\!\left[-R_p \mid -R_p > \text{VaR}_\alpha\right] = \frac{1}{1-\alpha} \int_\alpha^1 \text{VaR}_u \, du$$

ES is a coherent risk measure (Artzner et al. 1999) — it satisfies subadditivity ($\text{ES}(A+B) \leq \text{ES}(A) + \text{ES}(B)$), while VaR does not. Basel III/IV (FRTB) requires ES at 97.5% for internal models rather than VaR at 99%, precisely because ES captures the tail distribution rather than just the cutoff point.

6.2 Historical ES computation

The empirical estimate uses the same non-overlapping windows as historical VaR, then averages over the tail:

$$\widehat{\text{ES}}_\alpha = -\frac{1}{|\mathcal{T}|} \sum_{t \in \mathcal{T}} R_p^{(t)}, \quad \mathcal{T} = \{t : R_p^{(t)} \leq \hat q_{1-\alpha}\}$$

Key property verified in tests: $\text{ES}_\alpha \geq \text{VaR}_\alpha$ always. ES is at least as large as VaR — ES is the average of the worst outcomes, VaR is just the boundary. If this inequality fails, there is a bug in implementation.

Drishti flags the ES estimate as unstable if fewer than 30 tail observations contribute to the average.

7. Component VaR & Marginal Contributions

Market Risk

7.1 Marginal VaR

Under the parametric (delta-normal) framework, the portfolio VaR is:

$$\text{VaR}_p = z_\alpha \cdot \sigma_p = z_\alpha \sqrt{\mathbf{w}^\top \boldsymbol{\Sigma} \mathbf{w}}$$

The marginal VaR of asset $i$ is the sensitivity of portfolio VaR to a small change in weight $w_i$:

Marginal VaR
$$\text{MVaR}_i = \frac{\partial \text{VaR}_p}{\partial w_i} = z_\alpha \cdot \frac{(\boldsymbol{\Sigma}\mathbf{w})_i}{\sigma_p}$$

where $(\boldsymbol{\Sigma}\mathbf{w})_i$ is the $i$-th element of the covariance vector between asset $i$ and the portfolio.

7.2 Component VaR

Component VaR is the dollar contribution of asset $i$ to total portfolio VaR:

Component VaR (Euler decomposition)
$$\text{CVaR}_i = w_i \cdot \text{MVaR}_i = w_i \cdot z_\alpha \cdot \frac{(\boldsymbol{\Sigma}\mathbf{w})_i}{\sigma_p}$$

This is Euler's theorem for homogeneous functions of degree 1: since $\text{VaR}_p(\lambda \mathbf{w}) = \lambda \cdot \text{VaR}_p(\mathbf{w})$, the component VaRs sum exactly to the total:

$$\sum_{i=1}^n \text{CVaR}_i = \mathbf{w}^\top (\boldsymbol{\Sigma}\mathbf{w}) \cdot \frac{z_\alpha}{\sigma_p} = \sigma_p \cdot z_\alpha = \text{VaR}_p$$

Additivity property: Component VaRs always sum exactly to total parametric VaR. This is verified in test_component_var_sums_to_parametric_total.

The VaR share of asset $i$ is $\text{CVaR}_i / \text{VaR}_p$ — the fraction of total risk driven by that position. An asset with a large weight but low correlation to the rest of the portfolio may have a small VaR share; a small position that is highly correlated with the rest can have an outsized share.

8. Maximum Drawdown

Market Risk

Drawdown measures the decline from a historical peak in the cumulative value (or NAV) of the portfolio. Let $V_t = V_0 \prod_{s=1}^t (1+R_{p,s})$ be the portfolio value path. The running maximum is:

$$M_t = \max_{s \leq t} V_s$$

The drawdown at time $t$ is:

$$DD_t = \frac{V_t - M_t}{M_t} \leq 0$$

The maximum drawdown over the sample is $\text{MDD} = \min_t DD_t$ — the deepest trough relative to any prior peak. A drawdown episode is a period where $DD_t \leq -\delta$ for some threshold $\delta$ (Drishti uses $\delta = 10\%$ for the events study).

Recovery date is the first $t^* > t_{\text{trough}}$ at which $V_{t^*} \geq M_{t_{\text{peak}}}$, i.e. the portfolio returns to its prior peak.

9. Stress Testing

Market Risk

Historical stress scenarios replay actual market shocks and apply the observed factor moves to the current portfolio. For each scenario $s$ (COVID, rate shock, crude crash, INR crisis, election), the scenario return is:

$$R_p^{(s)} = \sum_{i=1}^n w_i \cdot \Delta R_i^{(s)}$$

where $\Delta R_i^{(s)}$ is the total return of asset $i$ over the scenario window (e.g. NIFTY fell 38.4% during COVID). The five scenarios in Drishti are calibrated to Indian market events and represent tail-risk scenarios that VaR models may not capture (since they are single point-in-time events, not samples from a stationary distribution).

Why stress test alongside VaR? VaR is backward-looking over a training window. Stress tests ask "what if the future repeats a specific historical disaster?" They are complementary: VaR captures typical tail risk; stress scenarios capture named tail events that may not appear in the training sample.

10. Kupiec Proportion-of-Failures (POF) Test

Backtesting

10.1 Motivation

A VaR model at confidence $\alpha$ should produce violations (actual loss exceeding VaR) with probability $p = 1 - \alpha$ on any given day. The Kupiec (1995) POF test tests whether the observed violation rate $\hat p = V/T$ is statistically consistent with the expected rate $p$, where $V$ is the number of violations in $T$ observations.

10.2 The likelihood ratio test

Under the null hypothesis $H_0: p_{\text{true}} = p$, violations are i.i.d. Bernoulli$(p)$. The log-likelihood under the null is:

$$\ell_0 = (T-V)\log(1-p) + V\log(p)$$

Under the alternative (violations occur at the true rate $\hat p = V/T$):

$$\ell_1 = (T-V)\log\!\left(1-\frac{V}{T}\right) + V\log\!\left(\frac{V}{T}\right)$$

The likelihood ratio statistic:

Kupiec LR statistic
$$LR_{\text{POF}} = -2(\ell_0 - \ell_1) = -2\!\left[(T-V)\log\frac{1-p}{1-\hat p} + V\log\frac{p}{\hat p}\right] \xrightarrow{d} \chi^2_1$$

Under $H_0$, the statistic is asymptotically $\chi^2$ with 1 degree of freedom. We reject the VaR model (at 5% level) when $LR_{\text{POF}} > \chi^2_{1,0.95} = 3.84$, equivalently when the $p$-value $< 0.05$.

Intuition: At 99% confidence over 252 trading days, we expect $252 \times 0.01 \approx 2.52$ violations. Observing 0 or ≥8 violations would be statistically unusual. The test formalizes this intuition.

11. Christoffersen Independence Test

Backtesting

11.1 Motivation

Kupiec only tests the number of violations, not their timing. A model could pass Kupiec while producing violations that cluster together — occurring on consecutive days of market stress. Christoffersen (1998) tests whether violations are serially independent.

11.2 The transition-probability approach

Let $I_t \in \{0,1\}$ be the violation indicator on day $t$. Define transition counts from the violation series:

$$n_{ij} = |\{t : I_{t-1}=i,\; I_t=j\}|, \quad i,j \in \{0,1\}$$

The key quantities are the conditional probabilities of a violation given the previous day's state:

$$\hat\pi_{01} = \frac{n_{01}}{n_{00}+n_{01}}, \quad \hat\pi_{11} = \frac{n_{11}}{n_{10}+n_{11}}$$ $$\hat\pi = \frac{n_{01}+n_{11}}{n_{00}+n_{01}+n_{10}+n_{11}}$$

Under $H_0$ (independence): $\pi_{01} = \pi_{11} = \pi$. Under $H_1$ (dependence / clustering): $\pi_{11} > \pi_{01}$.

11.3 The LR statistic

Christoffersen independence LR
$$LR_{\text{ind}} = -2\left[(n_{00}+n_{10})\log(1-\hat\pi) + (n_{01}+n_{11})\log\hat\pi \right.$$ $$\left. - n_{00}\log(1-\hat\pi_{01}) - n_{01}\log\hat\pi_{01} - n_{10}\log(1-\hat\pi_{11}) - n_{11}\log\hat\pi_{11}\right]$$ $$\xrightarrow{d} \chi^2_1 \text{ under } H_0$$

Bug fixed (Drishti v2): The denominator of $\hat\pi$ must be the total number of transitions (pairs), not the total number of observations. Using $T$ instead of $n_{00}+n_{01}+n_{10}+n_{11}$ inflates $\hat\pi$ and biases the test. The clustering narrative is also gated: it appears only when the test rejects independence (and $\hat\pi_{11} > \hat\pi_{01}$), not otherwise.

12. ADF Unit Root Test

Econometric Diagnostics

Before fitting GARCH or running Granger causality, we verify that the return series is stationary (no unit root). The Augmented Dickey-Fuller (ADF) test regression is:

$$\Delta r_t = \mu + \rho r_{t-1} + \sum_{j=1}^{p} c_j \Delta r_{t-j} + \varepsilon_t$$

$H_0: \rho = 0$ (unit root; non-stationary). The ADF $t$-statistic for $\hat\rho$ is compared against the Dickey-Fuller critical values (not standard $t$-critical values). The lag order $p$ is chosen by AIC.

For financial return series (as opposed to price levels), the ADF test strongly rejects the unit root ($p < 0.001$ typically) — returns are mean-reverting and stationary. This is a first-step sanity check before the rest of the diagnostics ladder.

13. Ljung-Box Test for Autocorrelation

Econometric Diagnostics

The Ljung-Box test checks whether the first $m$ autocorrelations of a series are jointly zero:

$$Q(m) = T(T+2)\sum_{k=1}^m \frac{\hat\rho_k^2}{T-k} \xrightarrow{d} \chi^2_m \text{ under } H_0$$

where $\hat\rho_k$ is the sample autocorrelation at lag $k$. $H_0$: all autocorrelations up to lag $m$ are zero (no serial correlation).

Drishti applies Ljung-Box to both raw returns (to detect autocorrelation / momentum) and to GARCH-standardized residuals $\hat z_t = r_t/\hat\sigma_t$ (to check whether GARCH has removed the ARCH structure). Ideally: $Q(\text{raw}) > Q(\hat z)$ — GARCH should eliminate dependence.

14. ARCH-LM Test for Heteroskedasticity

Econometric Diagnostics

Before fitting GARCH, we test whether volatility clustering (ARCH effects) is actually present. The Engle (1982) ARCH-LM test regresses squared returns on their own lags:

$$r_t^2 = a_0 + a_1 r_{t-1}^2 + \cdots + a_q r_{t-q}^2 + \nu_t$$

The test statistic is $LM = TR^2$ (from the auxiliary regression), asymptotically $\chi^2_q$ under $H_0: a_1=\cdots=a_q=0$ (no ARCH effects).

Interpretation in Drishti:

  • If ARCH-LM rejects $H_0$ → GARCH/ARCH model is justified; proceed to order scan.
  • If ARCH-LM fails to reject → returns are (conditionally) homoskedastic; parametric VaR may be adequate.

15. GARCH Order Scan (BIC Comparison)

Econometric Diagnostics

Once ARCH effects are confirmed, we fit four competing volatility models and compare by BIC to justify our GARCH(1,1) choice:

ModelSpecificationExtra parameter
garch_11GARCH(1,1): $\sigma_t^2 = \omega + \alpha r_{t-1}^2 + \beta\sigma_{t-1}^2$—
garch_12GARCH(1,2): adds second ARCH lag $\alpha_2 r_{t-2}^2$$\alpha_2$
garch_21GARCH(2,1): adds second GARCH lag $\beta_2\sigma_{t-2}^2$$\beta_2$
gjr_11GJR-GARCH(1,1): asymmetric; $\sigma_t^2 = \omega + (\alpha+\gamma\mathbf{1}_{r_{t-1}<0})r_{t-1}^2 + \beta\sigma_{t-1}^2$$\gamma$ (leverage)

The GJR model captures the leverage effect: negative shocks ($r_{t-1} < 0$) increase volatility more than positive shocks of the same magnitude ($\gamma > 0$ for equities). The BIC is:

$$\text{BIC} = -2\hat\ell + k\log T$$

Lower BIC is better. If GARCH(1,1) has the lowest BIC among the four candidates, it is the parsimonious choice. If GJR-GARCH wins, the asymmetry is economically meaningful (leverage effect).

16. Engle-Sheppard CCC Test

Econometric Diagnostics

16.1 Motivation

Before fitting DCC-GARCH (which assumes time-varying correlations), we test whether correlations are actually constant. The Engle and Sheppard (2001) test has $H_0$: Constant Conditional Correlation (CCC). Rejection justifies fitting DCC.

16.2 Procedure

GARCH-standardize each series: fit GARCH(1,1) to each return series $r_{i,t}$ and compute $z_{i,t} = r_{i,t}/\hat\sigma_{i,t}$.
CCC-whiten: compute the unconditional correlation matrix $\hat R = \text{corr}(\mathbf{Z})$, then $\mathbf{U}_t = \hat R^{-1/2} \mathbf{z}_t$. Under CCC, $\mathbf{U}_t$ should be spherical.
Test regression: for the mean off-diagonal product deviation $y_t$ at each time step, regress on its own lags: $$y_t = c + \sum_{j=1}^m \delta_j y_{t-j} + \varepsilon_t$$
Wald test: $H_0: \delta_1 = \cdots = \delta_m = 0$ (no dynamics in joint second moments). The Wald statistic $\sim \chi^2_m$ under $H_0$. Rejection ⟹ correlations are time-varying ⟹ DCC is appropriate.

17. GARCH(1,1) Estimation

Volatility Models

17.1 Quasi-maximum likelihood

GARCH parameters $\boldsymbol\theta = (\omega, \alpha, \beta)$ are estimated by maximizing the Gaussian quasi-log-likelihood:

GARCH log-likelihood
$$\mathcal{L}(\boldsymbol\theta) = -\frac{1}{2}\sum_{t=1}^{T}\left[\log(2\pi) + \log\sigma_t^2 + \frac{r_t^2}{\sigma_t^2}\right]$$

where $\sigma_t^2$ is recursively computed as a function of $\boldsymbol\theta$ and the data. Even if returns are not truly Gaussian, the QMLE estimator is consistent and asymptotically normal under mild regularity conditions (Bollerslev & Wooldridge 1992).

In Drishti, estimation is performed by the arch Python library which uses SLSQP optimization with analytical gradients. Returns are scaled to percentage units (×100) before fitting to improve numerical conditioning, then rescaled back.

18. Hidden Markov Model for Regime Detection

Volatility Models

18.1 The two-state model

A Gaussian HMM assumes the observed feature vector $\mathbf{x}_t$ (rolling volatility, absolute returns, India VIX level and change) is generated by a hidden state $s_t \in \{0, 1\}$ that follows a first-order Markov chain:

HMM generative model
$$\Pr(s_t = j \mid s_{t-1} = i) = A_{ij} \quad \text{(transition matrix)}$$ $$\mathbf{x}_t \mid s_t = k \sim \mathcal{N}(\boldsymbol\mu_k, \boldsymbol\Sigma_k) \quad \text{(emission)}$$

The two states are canonically labeled: State 0 = low-vol (bull), State 1 = high-vol (bear), determined after fitting by the emission mean on the rolling-volatility feature. This prevents the label-switching problem where monthly refits arbitrarily swap state 0 and state 1.

18.2 The Baum-Welch EM algorithm

Parameters $\boldsymbol\theta = \{A, \{\boldsymbol\mu_k\}, \{\boldsymbol\Sigma_k\}\}$ are estimated by Expectation-Maximization:

  • E-step (Forward-Backward): compute posterior state probabilities $\gamma_t(k) = \Pr(s_t = k \mid \mathbf{X}, \boldsymbol\theta^{(\text{old})})$ using the forward $\alpha_t$ and backward $\beta_t$ recursions.
  • M-step: update parameters to maximize the expected complete-data log-likelihood: $$\hat{\boldsymbol\mu}_k = \frac{\sum_t \gamma_t(k)\mathbf{x}_t}{\sum_t \gamma_t(k)}, \quad \hat{\boldsymbol\Sigma}_k = \frac{\sum_t \gamma_t(k)(\mathbf{x}_t-\hat{\boldsymbol\mu}_k)(\mathbf{x}_t-\hat{\boldsymbol\mu}_k)^\top}{\sum_t \gamma_t(k)}$$

18.3 Walk-forward fitting

To avoid look-ahead bias, Drishti uses a walk-forward approach: the HMM is fit on all data up to time $T_{\text{train}}$, then used to predict regimes for the next 21 trading days (1 month), then the window expands by 21 days and refitting occurs. Only OOS predictions are kept. Minimum training window: 252 trading days.

18.4 Regime-conditioned VaR

VaR is computed separately for each regime using the historical returns assigned to that regime. In the high-vol state, VaR is significantly higher — capturing the fact that the tail distribution is regime-dependent, not constant.

19. DCC-GARCH (Dynamic Conditional Correlations)

Volatility Models

DCC-GARCH (Engle 2002) is a two-step estimator that models time-varying correlations between multiple series without estimating a full multivariate GARCH model (which would be computationally infeasible for $k > 5$ series).

19.1 Step 1 — Univariate GARCH

Fit GARCH(1,1) independently to each return series $r_{i,t}$, $i = 1, \ldots, k$. Extract standardized residuals:

$$\hat z_{i,t} = \frac{r_{i,t}}{\hat\sigma_{i,t}}$$

Gather into matrix $\mathbf{Z} = [\hat z_1, \ldots, \hat z_k] \in \mathbb{R}^{T \times k}$.

19.2 Step 2 — DCC correlation dynamics

The unconditional covariance of standardized residuals is $\bar{Q} = T^{-1}\mathbf{Z}^\top\mathbf{Z}$. The DCC recursion:

DCC Q-process
$$Q_t = (1-\alpha-\beta)\bar{Q} + \alpha \mathbf{z}_{t-1}\mathbf{z}_{t-1}^\top + \beta Q_{t-1}$$

$Q_t$ is a pseudo-correlation matrix. The actual conditional correlation matrix $R_t$ is obtained by normalizing:

$$R_t = \text{diag}(Q_t)^{-1/2} Q_t\, \text{diag}(Q_t)^{-1/2}$$

so $R_{t,ii} = 1$ (diagonal ones). The conditional covariance matrix of returns is $H_t = D_t R_t D_t$ where $D_t = \text{diag}(\hat\sigma_{1,t}, \ldots, \hat\sigma_{k,t})$.

19.3 DCC log-likelihood

Parameters $(\alpha, \beta)$ are estimated by maximizing the DCC contribution to the log-likelihood (after profiling out Step 1 parameters):

$$\ell_{\text{DCC}} = -\frac{1}{2}\sum_{t=1}^T \left[\log|R_t| + \mathbf{z}_t^\top R_t^{-1} \mathbf{z}_t - \mathbf{z}_t^\top \mathbf{z}_t\right]$$

Stationarity constraint: $\alpha + \beta < 1$. Optimization via SLSQP with analytical gradient.

19.4 Interpretation

The time-varying correlation $R_{t,ij}$ between sector $i$ and commodity $j$ is Drishti's primary evidence for spillover. During COVID-2020, correlations between all sectors and commodities (particularly crude oil) spiked simultaneously — a visible signature of systemic risk that constant-correlation models miss.

20. ADCC — Asymmetric DCC

Volatility Models

Cappiello, Engle & Sheppard (2006) extend DCC to capture the leverage effect in correlations: negative joint shocks tend to increase correlations more than positive shocks of equal magnitude. This is observed in equity markets — co-crashes are more common than co-rallies.

20.1 The ADCC Q-recursion

Define the "negative part" of the shock: $\mathbf{n}_t = \mathbf{z}_t \odot \mathbf{1}[\mathbf{z}_t < 0]$ (elementwise, zero for positive shocks). Let $\bar{N} = T^{-1}\sum_t \mathbf{n}_t \mathbf{n}_t^\top$. The ADCC recursion is:

ADCC Q-process (Cappiello-Engle-Sheppard 2006)
$$Q_t = (1-a-b)\bar{Q} - g\bar{N} + a\mathbf{z}_{t-1}\mathbf{z}_{t-1}^\top + b Q_{t-1} + g\mathbf{n}_{t-1}\mathbf{n}_{t-1}^\top$$

The $-g\bar{N}$ intercept correction ensures $\mathbb{E}[Q_t] = \bar{Q}$ in steady state (mean-reversion to the unconditional correlation). Stationarity sufficient condition: $a + b + g < 1$.

If $g > 0$: negative joint shocks ($\mathbf{z}_{t-1} < 0$) increase $Q_t$ by more than positive shocks → correlation increases more during crashes than rallies. This is the asymmetric correlation leverage effect.

20.2 Estimation

Three parameters $(a, b, g)$ are jointly estimated by maximizing the same DCC log-likelihood as above with the ADCC $Q_t$ recursion. If the joint optimizer fails (common on real data due to a flat likelihood surface), Drishti falls back to profiling $g$ over a coarse grid $\{0, 0.05, 0.10, 0.15\}$ and re-optimizing $(a, b)$ at each point — a more robust but slower procedure.

21. Diebold-Yilmaz Connectedness Index

Spillover Research

21.1 The VAR model

For a $k$-dimensional system of returns $\mathbf{r}_t = (r_{1,t}, \ldots, r_{k,t})^\top$, fit a Vector Autoregression of order $p$ (chosen by AIC, capped at $p=5$):

$$\mathbf{r}_t = \sum_{j=1}^p A_j \mathbf{r}_{t-j} + \boldsymbol\varepsilon_t, \quad \boldsymbol\varepsilon_t \sim \mathcal{N}(\mathbf{0}, \Sigma)$$

The moving-average representation $\mathbf{r}_t = \sum_{h=0}^\infty \Psi_h \boldsymbol\varepsilon_{t-h}$ with $\Psi_0 = I_k$ and $\Psi_h = \sum_{j=1}^{\min(h,p)} A_j \Psi_{h-j}$ decomposes variance through impulse-response functions.

21.2 Generalized FEVD (Pesaran-Shin)

Classical FEVD uses Cholesky decomposition and is order-dependent — the result changes if you reorder variables. Pesaran-Shin (1998) generalized FEVD is order-invariant:

GFEVD (H-step)
$$\tilde\theta_{ij}(H) = \frac{\sigma_{jj}^{-1} \sum_{h=0}^{H-1} (\mathbf{e}_i^\top \Psi_h \Sigma \mathbf{e}_j)^2}{\sum_{h=0}^{H-1} \mathbf{e}_i^\top \Psi_h \Sigma \Psi_h^\top \mathbf{e}_i}$$

where $\sigma_{jj}$ is the $j$-th diagonal element of $\Sigma$, and $\mathbf{e}_i$ is the $i$-th standard basis vector. Row-normalize to ensure each row sums to 1:

$$\theta_{ij}(H) = \frac{\tilde\theta_{ij}(H)}{\sum_{j=1}^k \tilde\theta_{ij}(H)}$$

21.3 The spillover table

From $\Theta = [\theta_{ij}]$, Diebold-Yilmaz (2012) define:

MeasureFormulaInterpretation
Total connectedness $C = \frac{\sum_{i\neq j}\theta_{ij}}{k} \times 100\%$ % of forecast error variance from cross-variable shocks (industry-wide integration)
Directional: to $i$ $S_{\to i} = \frac{\sum_{j\neq i}\theta_{ji}}{k}\times 100\%$ How much variance $i$ contributes to others
Directional: from $i$ $S_{i\to} = \frac{\sum_{j\neq i}\theta_{ij}}{k}\times 100\%$ How much of $i$'s variance comes from others
Net spillover $S_i^{\text{net}} = S_{\to i} - S_{i\to}$ Positive = net transmitter; Negative = net receiver

Drishti finding (v2 real data): Large-cap total connectedness ≈ 67% — highly integrated. Mid-cap ≈ 43% — more idiosyncratic. Commodities (Brent, Metals) are net transmitters to Indian equity sectors.

21.4 Rolling connectedness

A 200-day rolling window (step = 21 days) produces the time-series of total connectedness $C_t$. Spikes in $C_t$ identify systemic risk events — the 2020 COVID crash and the 2022 Ukraine/commodity shock are clearly visible as sharp increases in cross-market variance transmission.

22. Time-Series Information Coefficient (IC)

Factor Research

22.1 Why time-series IC, not cross-sectional

A commodity return (e.g. Brent crude) at time $t$ is a scalar — it is the same for all $N$ stocks at that moment. Cross-sectional rank correlation between a scalar and a cross-section is undefined or trivially degenerate. The correct specification is time-series IC: does the commodity factor lagged by $\ell$ days predict sector returns at time $t$?

22.2 Rolling Pearson IC

For factor $f$ and target sector return $y$, the IC at lag $\ell$ over a rolling window of $W = 63$ trading days is:

Time-series IC
$$\text{IC}_{t,\ell} = \text{Corr}\!\left(\{f_{s-\ell}\}_{s=t-W+1}^{t},\; \{y_s\}_{s=t-W+1}^{t}\right)$$

The mean IC over all time periods and the ICIR (IC information ratio) are:

$$\overline{\text{IC}} = \frac{1}{T-W}\sum_t \text{IC}_{t,\ell}, \qquad \text{ICIR} = \frac{\overline{\text{IC}}}{\text{std}(\text{IC}_{t,\ell})}$$

22.3 HAC standard errors for significance

Naively treating the rolling IC series as i.i.d. inflates significance: adjacent windows share $W-1$ observations (roughly an MA($W-1$) dependence). Drishti uses Newey-West heteroskedasticity-and-autocorrelation-consistent (HAC) standard errors with truncation lag $= W$:

$$\hat V_{\text{HAC}} = \hat\sigma_0^2 + 2\sum_{j=1}^W \left(1 - \frac{j}{W+1}\right)\hat\sigma_j^2$$

where $\hat\sigma_j^2 = T^{-1}\sum_t \hat u_t \hat u_{t-j}$ are sample autocovariances of the regression residuals. The HAC $t$-statistic $= \overline{\text{IC}} / \sqrt{\hat V_{\text{HAC}}/T}$ is then compared to $\mathcal{N}(0,1)$.

23. Granger Causality

Factor Research

23.1 The concept

Variable $X$ Granger-causes $Y$ if past values of $X$ improve the forecast of $Y$ beyond what is achievable from $Y$'s own lags alone. This is a predictive concept — not physical causality, but useful for factor signal research.

23.2 The F-test formulation

At lag $\ell$, the test compares two regressions:

Restricted (no Granger causality)
$$y_t = c + \sum_{j=1}^\ell a_j y_{t-j} + u_t^R$$
Unrestricted (with factor lags)
$$y_t = c + \sum_{j=1}^\ell a_j y_{t-j} + \sum_{j=1}^\ell b_j x_{t-j} + u_t^U$$

The Granger F-statistic tests $H_0: b_1 = \cdots = b_\ell = 0$:

$$F = \frac{(SSR_R - SSR_U)/\ell}{SSR_U/(T - 2\ell - 1)} \sim F(\ell,\, T-2\ell-1) \text{ under } H_0$$

23.3 Weekly frequency

Daily Granger tests suffer from microstructure noise and false positives. Drishti compounds daily returns into weekly returns (W-FRI bucket) before testing:

$$R_{\text{weekly}} = \prod_{t \in \text{week}} (1 + r_t) - 1$$

Tests are run at lags 1–13 weeks (≈ 3 months). This is the economically relevant horizon for commodity-to-equity propagation (supply chain adjustments, inflation pass-through).

23.4 AIC-summary

Running Granger up to lag 13 produces 13 test results per (factor, sector) pair. The AIC from the unrestricted regression $AIC = -2\hat\ell + 2k$ identifies the optimal lag; summarize_granger_aic() retains only the min-AIC row per pair for reporting.

24. Benjamini-Hochberg FDR Correction

Factor Research

24.1 The multiple testing problem

Drishti tests ≈ 200+ (factor × sector × lag) combinations for IC significance and ≈ 150+ pairs for Granger causality. At $\alpha = 0.05$ per test, we expect $0.05 \times 200 = 10$ false positives by chance. Reporting all $p < 0.05$ results would be misleading.

24.2 Benjamini-Hochberg procedure

Given $m$ tests with $p$-values $p_{(1)} \leq p_{(2)} \leq \cdots \leq p_{(m)}$, reject the $k$ hypotheses corresponding to the $k^* = \max\{k : p_{(k)} \leq \frac{k}{m}\alpha\}$ smallest $p$-values:

BH rejection threshold
$$k^* = \max\left\{k \in \{1,\ldots,m\} : p_{(k)} \leq \frac{k}{m} \cdot \alpha\right\}$$

Reject $H_{(1)}, \ldots, H_{(k^*)}$ at level $\alpha$. BH controls the False Discovery Rate (FDR) — the expected proportion of false discoveries among all rejections — at level $\alpha$ under independence, and at level $\alpha \cdot m_0/m$ under positive dependence (where $m_0$ is the true number of null hypotheses).

Drishti reports both the uncorrected $p$-value significance (significant) and the BH-corrected flag (bh_significant). The BH flag is the one used for reporting and charting — raw $p$-values are displayed for transparency.

25. Walk-Forward Out-of-Sample Sharpe

Factor Research

For each (factor, sector) pair that passes BH correction, Drishti estimates the OOS Sharpe ratio via walk-forward cross-validation to avoid overfitting:

Train window: 252 trading days. OOS step: 21 days (monthly).
Signal construction: at each refit point, use the IC-optimal lag from the training window to create a lagged factor signal.
OOS return: sign(signal) × sector_return over the next 21 days (long when signal positive, flat or short otherwise).
OOS Sharpe: $S = \frac{\bar R_{\text{OOS}}}{\hat\sigma_{\text{OOS}}} \cdot \sqrt{252}$ — annualized over all OOS periods.

The result is a heatmap of OOS Sharpe per (factor × sector) — a measure of how consistently the commodity factor predicts sector returns in unseen data, not just in-sample.

26. Bull/Bear Regime Classification (20% Rule)

Research Studies

The classical 20% rule defines a bear market as a decline of ≥ 20% from a prior peak, and a bull market as a rally of ≥ 20% from a prior trough. This is the NBER turning-point methodology adapted for equity indices.

State machine
$$\text{State} = \begin{cases}\text{bear} & \text{if } V_t \leq V_{\text{peak}} \cdot (1-0.20) \text{ from bull state} \\ \text{bull} & \text{if } V_t \geq V_{\text{trough}} \cdot (1+0.20) \text{ from bear state}\end{cases}$$

Peak and trough are tracked as running max/min within each state. The 20% threshold prevents false signals from noise — small corrections do not trigger state changes.

26.1 Regime signs table

For each regime, Drishti computes a statistical summary of return characteristics:

StatisticFormula
Mean daily return$\bar r = T_s^{-1}\sum_{t\in s} r_t$
Annualized volatility$\hat\sigma_{\text{ann}} = \hat\sigma \cdot \sqrt{252}$
Skewness$\text{Skew} = T_s^{-1}\sum (r_t - \bar r)^3 / \hat\sigma^3$
Worst day$\min_{t\in s} r_t$
% Up days$T_s^{-1}|\{t\in s: r_t > 0\}|$

26.2 HMM overlay

The 20% rule is deterministic and lagging (you only know a bear market started 20% after the peak). The HMM overlay provides a real-time probabilistic view: $\Pr(\text{high-vol state}_t)$ from the walk-forward HMM. The two views are complementary and both displayed on the Regimes tab.

27. Market Shock Events Study

Research Studies

27.1 Drawdown episode detection algorithm

An episode is detected when the running drawdown from the current peak crosses the threshold $\delta = 10\%$:

Compute running maximum $M_t = \max_{s\leq t} P_s$ and drawdown $DD_t = P_t/M_t - 1$.
Enter episode at the first $t$ where $DD_t \leq -\delta$. Record $t_{\text{start}}$.
Within the episode, track the trough date $t^* = \arg\min_{t \geq t_{\text{start}}} DD_t$.
Exit episode when $DD_t \geq 0$ (full recovery). Record recovery date.
Peak date = the last date at which $P_t = M_{t^*}$ (the pre-crash high), found using relative tolerance $|P_t - M_{t^*}| < M_{t^*} \cdot 10^{-6}$ for float safety.

27.2 Label matching

The 16 curated labels (dot-com through 2026 tariff shock) are matched to detected episodes by window overlap: label $l$ is attached to episode $e$ if $t_{\text{peak}} \leq W_l^{\text{end}}$ and $(t_{\text{recovery}} \text{ or } t_{\text{trough}}) \geq W_l^{\text{start}}$. Unmatched episodes are labeled "Unlabeled episode."

27.3 Statistical levels

Descriptive statistics of drawdown depths serve as a historical reference guide:

$$\text{Median fall} = \text{Median}\{|d_e|\}, \quad P_{75} = \text{Quantile}_{75\%}\{|d_e|\}, \quad P_{90} = \text{Quantile}_{90\%}\{|d_e|\}$$

These are purely descriptive and explicitly labeled as "not predictions." A separate practitioner appendix contains technical analysis heuristics (round-number levels, 200-DMA distance at trough), clearly flagged as heuristics rather than statistics. The methodological separation between statistics and heuristics is a deliberate design choice.

28. XGBoost VaR Breach Classifier

Machine Learning

28.1 Task formulation

Predict whether tomorrow's portfolio return will exceed the historical 99% VaR (i.e., a VaR breach will occur) using today's information only. This is a binary classification problem:

$$Y_{t+1} = \mathbf{1}\!\left[R_{p,t+1} < -\text{VaR}_{0.99}\right]$$

No look-ahead: The target variable uses r.shift(-1) — tomorrow's return. All features are based on information available today (or earlier). SMOTE for class imbalance is applied after train/test split — never on the test set.

28.2 Feature engineering

FeatureConstructionEconomic rationale
brent_lag1Yesterday's Brent crude returnCommodity shock → equity tail risk
gold_lag1Yesterday's gold returnRisk-off signal
copper_lag1Yesterday's copper returnIndustrial demand signal
rolling_vol_55-day rolling std of portfolio returnsRecent volatility clustering
rolling_vol_2121-day rolling stdMonthly vol regime

28.3 Class imbalance handling

VaR breaches at 99% confidence occur ≈ 1% of the time — a severe class imbalance. XGBoost handles this via scale_pos_weight = (number of negatives) / (number of positives), which up-weights the rare positive class in the gradient computation. SMOTE (synthetic over-sampling) was evaluated but rejected in favor of scale_pos_weight to avoid distributional contamination of the test set.

28.4 XGBoost loss function

XGBoost minimizes a regularized second-order approximation of the binary cross-entropy loss:

$$\mathcal{L} = \sum_i \left[\ell(y_i, \hat y_i) + \Omega(f_k)\right]$$ $$\Omega(f) = \gamma T + \frac{1}{2}\lambda\|\mathbf{w}\|^2$$

where $T$ is the number of leaves and $\mathbf{w}$ are leaf weights. The model is a sum of $K$ CART trees fitted greedily on the negative gradient (pseudo-residuals) of the loss. The output probability is $\hat p = \sigma(\hat y)$ where $\sigma$ is the sigmoid function.

28.5 Evaluation

The primary metric is AUC-PR (area under the precision-recall curve), not AUC-ROC, because the positive class is rare. A model that predicts all zeros achieves AUC-ROC ≈ 0.99 but AUC-PR ≈ 0.01 — useless for the actual task. The dashboard shows breach probability as a gauge and feature importances as a bar chart.

29. FinBERT Sentiment Analysis

Machine Learning

29.1 Architecture

FinBERT (Araci 2019; ProsusAI/finbert) is a BERT-base model fine-tuned on financial news corpora for three-class sentiment: positive, negative, neutral. The underlying architecture is a 12-layer Transformer encoder with 768-dimensional hidden states:

$$\mathbf{h}_t = \text{Transformer}\!\left(\mathbf{e}_t\right), \quad \hat{\mathbf{p}} = \text{softmax}\!\left(W\mathbf{h}_{[\text{CLS}]}\right) \in \mathbb{R}^3$$

where $\mathbf{e}_t$ is the input embedding sequence and $\mathbf{h}_{[\text{CLS}]}$ is the representation of the special [CLS] token.

29.2 Pipeline

Drishti fetches RSS headlines from 6 Indian financial sources (ET Markets, NSE, Moneycontrol, Business Standard, LiveMint, Hindu BusinessLine), scores each headline through FinBERT, and caches results with a 2-hour staleness window. A composite sentiment score is injected into the deterministic risk memo as context.

29.3 Module-level pipeline cache

Loading the FinBERT model from disk takes ≈ 5–15 seconds. Drishti caches the pipeline object at module level so the model is loaded once per process, not once per request. The /api/research/news/refresh route offloads scoring to a thread pool (asyncio.to_thread) to avoid blocking FastAPI's event loop.

31. Risk-Adjusted Performance: Sharpe, Treynor, Jensen

v3 — Performance (FRM Wk3)

Three complementary risk-adjusted return measures, each dividing excess return by a different risk denominator. With daily returns $r_t$, market $m_t$, and an annualized risk-free rate $r_f$ (converted to per-period $r_f/252$):

Sharpe — excess return per unit of total volatility, annualized: $$S = \frac{\overline{(r-r_f/252)}}{\hat\sigma_{r-r_f/252}}\,\sqrt{252}$$

Beta and Treynor — excess return per unit of systematic risk: $$\beta = \frac{\mathrm{Cov}(r,m)}{\mathrm{Var}(m)}, \qquad T = \frac{\overline{r}\cdot 252 - r_f}{\beta}$$

Jensen's alpha — return above the CAPM prediction: $$\alpha_J = \overline{r}\cdot 252 - \big[\,r_f + \beta\,(\overline{m}\cdot 252 - r_f)\,\big]$$

Implemented in src/risk/performance.py; beta and the mean terms are computed on the index-aligned common sample. The risk-free rate is taken from the India 10Y G-Sec yield (GIND10YR), which is stored in percent and divided by 100 before use.

32. Markowitz Mean-Variance Efficient Frontier

v3 — Portfolio (FRM Wk1)

Given an annualized mean vector $\mu$ and covariance $\Sigma$, the minimum-variance portfolio solves $$\min_{w}\; w^\top \Sigma w \quad \text{s.t.}\quad \mathbf{1}^\top w = 1,\; (w \ge 0 \text{ if long-only}).$$

The efficient frontier traces the minimum variance for each target return $\mu^\star$ over $[\mu_{\text{mv}}, \max_i \mu_i]$: $$\min_{w}\; w^\top \Sigma w \quad \text{s.t.}\quad \mathbf{1}^\top w = 1,\; w^\top\mu = \mu^\star.$$

The tangency portfolio maximizes the Sharpe ratio $\frac{w^\top\mu - r_f}{\sqrt{w^\top\Sigma w}}$; the capital market line runs from $(0, r_f)$ through it.

Solved with SLSQP (src/portfolio/frontier.py). Because the long-only frontier variance is theoretically non-decreasing above the min-variance vertex, the first frontier point is anchored to the exact min-variance solution and a running-max clamp removes residual solver noise so the frontier is deterministically monotone. Diagnostic only — never a recommendation to hold the weights.

33. EWMA / RiskMetrics Volatility & Covariance

v3 — Volatility (FRM Wk7/8)

The RiskMetrics exponentially weighted moving average gives the conditional variance more weight to recent observations: $$h_t = \lambda\, h_{t-1} + (1-\lambda)\, r_{t-1}^2,$$ with $\lambda = 0.94$ for daily data (RiskMetrics; J.R. Varma committee for NSE). The multivariate analogue updates the covariance by the lagged outer product: $$\Sigma_t = \lambda\,\Sigma_{t-1} + (1-\lambda)\, r_{t-1} r_{t-1}^\top,$$ which is positive semi-definite by construction (a convex combination of a PSD seed and PSD rank-one terms). In src/risk/ewma.py.

34. EVT-VaR via Peaks-Over-Threshold (GPD)

v3 — Tail Risk (FRM Wk8)

Extreme Value Theory models the tail of the loss distribution ($L = -r$) above a high threshold $u$ with a Generalized Pareto Distribution. With $N_u$ exceedances out of $n$ losses and fitted shape $\xi$, scale $\beta$ (MLE), the McNeil–Frey tail estimators are: $$\mathrm{VaR}_q = u + \frac{\beta}{\xi}\!\left[\Big(\tfrac{n}{N_u}(1-q)\Big)^{-\xi} - 1\right], \qquad \mathrm{ES}_q = \frac{\mathrm{VaR}_q + \beta - \xi u}{1-\xi}\;\;(\xi<1).$$

Valid only for $q \ge$ the threshold quantile (the formula extrapolates into the tail). src/risk/evt.py guards against $q<$ threshold, too few exceedances, and an absent loss tail. Genuinely different from the Gaussian methods — it captures heavy tails parametrically.

35. Range-Based (Extreme-Value) Volatility Estimators

v3 — Volatility (FRM Wk7)

OHLC range estimators are several times more efficient than close-to-close standard deviation. With open $O$, high $H$, low $L$, close $C$ per bar:

Parkinson: $\;\hat\sigma^2_P = \dfrac{1}{4\ln 2}\,\overline{\big(\ln \tfrac{H}{L}\big)^2}$

Garman–Klass: $\;\hat\sigma^2_{GK} = \overline{\tfrac12\big(\ln\tfrac{H}{L}\big)^2 - (2\ln 2 - 1)\big(\ln\tfrac{C}{O}\big)^2}$

Rogers–Satchell (drift-independent): $\;\hat\sigma^2_{RS} = \overline{\ln\tfrac{H}{C}\ln\tfrac{H}{O} + \ln\tfrac{L}{C}\ln\tfrac{L}{O}}$

Each per-bar term is non-negative, so the estimators are well-defined. In src/risk/extreme_value_vol.py; fed by the dedicated OHLC pull (scripts/pull_ohlc_frtl.py).

36. Threshold Autoregression (TAR) for Regimes

v3 — Regimes (SAAPM Wk4)

A two-regime self-exciting TAR (Tsay/Tong) fits a separate AR($p$) on either side of a threshold $\tau$ on the delay-$d$ lagged value $y_{t-d}$: $$y_t = \begin{cases} \phi_0^{(1)} + \sum_{k=1}^p \phi_k^{(1)} y_{t-k} + \varepsilon_t, & y_{t-d} \le \tau \\[4pt] \phi_0^{(2)} + \sum_{k=1}^p \phi_k^{(2)} y_{t-k} + \varepsilon_t, & y_{t-d} > \tau \end{cases}$$

$\tau$ is chosen by grid search over the trimmed quantile band of $y_{t-d}$ to minimize the combined regime SSR (conditional least squares). Because the threshold is unidentified under the linear null (Davies' problem), significance is assessed by a bootstrap likelihood-ratio test, $\mathrm{LR} = n\ln(\mathrm{SSR}_0/\mathrm{SSR}_1)$, with the p-value from residual-bootstrap replicates under the linear AR. Applied to the detrended log-price (above-trend = bull, below = bear) and cross-checked against the HMM and 20%-rule regimes. In src/research/tar.py.

37. Johansen Cointegration & VECM

v3 — Multivariate (SAAPM Wk3/4)

Before differencing to returns, the Johansen trace test checks for long-run equilibrium relationships among the $I(1)$ log-levels. The cointegrating rank $r$ is the number of trace statistics exceeding their 95% critical values. If $r \ge 1$, a Vector Error Correction Model $$\Delta y_t = \Pi\, y_{t-1} + \sum_{i=1}^{k-1}\Gamma_i\,\Delta y_{t-i} + \varepsilon_t, \qquad \Pi = \alpha\beta^\top,\;\; \mathrm{rank}(\Pi)=r,$$ captures both the short-run dynamics and the error-correction toward equilibrium. This complements the VAR-on-returns spillover study, which would otherwise discard any long-run relationship. In src/research/cointegration.py (wrapping statsmodels).

38. Altman Z-Score Credit Screen

v3 — Credit (FRM Wk9)

Altman's (1968) discriminant score combines five fundamental ratios: $$Z = 1.2\,x_1 + 1.4\,x_2 + 3.3\,x_3 + 0.6\,x_4 + 1.0\,x_5,$$ where $x_1=\tfrac{\text{WC}}{\text{TA}}$, $x_2=\tfrac{\text{RE}}{\text{TA}}$, $x_3=\tfrac{\text{EBIT}}{\text{TA}}$, $x_4=\tfrac{\text{MVE}}{\text{TL}}$, $x_5=\tfrac{\text{Sales}}{\text{TA}}$. Zones: $Z>2.99$ safe, $1.81 \le Z \le 2.99$ grey, $Z<1.81$ distress. Implemented in src/research/credit.py and complemented by a reduced credit-health screen (leverage, ROA, ROE, market-cap/debt, cash-flow/debt) on the available cached fundamentals.

39. Amihud Illiquidity

v3 — Liquidity (FRM Wk9)

The Amihud (2002) ILLIQ measure is the average price impact per unit of traded value — the mean absolute return divided by daily currency volume (price × shares): $$\text{ILLIQ} = \frac{1}{D}\sum_{t=1}^{D} \frac{|r_t|}{P_t\,V_t}\times 10^6.$$ Higher values indicate thinner liquidity (more price impact per rupee traded). In src/research/liquidity.py; ranked across names and against market capitalization.

DCC over full multivariate GARCH BEKK, VECH BEKK: $O(k^2)$ parameters. VECH: no PSD guarantee. DCC: $k+2$ parameters; two-step consistent.
ADCC over symmetric DCC Symmetric DCC Leverage effect in correlations is empirically documented. ADCC adds one scalar $g$ with interpretable meaning.
Pesaran-Shin GFEVD over Cholesky FEVD Cholesky FEVD Cholesky FEVD depends on variable ordering — arbitrary for our use case. GFEVD is order-invariant.
Time-series IC (rolling Pearson) Cross-sectional rank IC A commodity return is a scalar at each $t$ — cross-sectional IC is undefined/degenerate. Time-series IC is the only valid specification.
HAC $t$-test for IC significance Naive ICIR $\times \sqrt{N}$ Rolling-window ICs share $\approx W-1$ observations. Naive test overstates significance by $\approx\sqrt{W}$. HAC corrects for overlap-induced autocorrelation.
Weekly Granger (not daily) Daily Granger Daily tests suffer from microstructure noise and inflated power. Weekly is the appropriate horizon for commodity propagation through supply chains.
BH FDR correction Bonferroni Bonferroni controls FWER and is overly conservative with 200+ tests. BH controls FDR, more appropriate for exploratory factor research.
20% rule + HMM dual regime view Only one method 20% rule: deterministic, lagging, investor-familiar. HMM: probabilistic, real-time, data-driven. Neither alone is sufficient.
Statistical levels separate from heuristics Mixed presentation Percentile levels (median/p75/p90 of historical drawdowns) are statistics. Round-number levels, 200-DMA distances are practitioner heuristics. Mixing them without labeling is methodologically dishonest.
scale_pos_weight over SMOTE SMOTE oversampling SMOTE applied to the full dataset before split contaminates the test set. scale_pos_weight re-weights gradient updates in training only, with no distributional leakage.
