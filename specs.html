1. Specification Overview

The system imports or simulates a Zerodha portfolio, normalizes holdings, fetches or loads historical market data, calculates market risk, augments the result with commodity spillover and news context, then presents results through a dashboard and AI copilot.

Import
Zerodha, CSV, or sample holdings.
Normalize
Symbols, quantities, prices, sectors, asset classes.
Enrich
Historical prices, factor series, RSS headlines.
Compute
VaR, ES, stress, contribution, spillover.
Explain
Risk narratives and MCP tool outputs.
Present
Dashboard, memo, and copilot answers.

4. Functional Requirements

ID Requirement Acceptance criteria
FR-01 Import portfolio from Zerodha, CSV, or sample fixture. At least one path works without external network during demo.
FR-02 Normalize securities into modeled equity/ETF and unsupported buckets. Unsupported instruments are visible and excluded from deep metrics.
FR-03 Fetch or load historical daily prices for modeled holdings. Risk engine receives aligned return matrix with missing data report.
FR-04 Calculate portfolio risk metrics. Dashboard shows VaR, ES, volatility, drawdown, concentration, and contribution.
FR-05 Run stress scenarios. Each scenario outputs estimated loss, affected sectors, and largest contributors.
FR-06 Analyze commodity spillover. System reports strongest factor, lag, direction, and confidence/significance indicator.
FR-07 Summarize RSS headline risk. System shows source, link, sector/ticker tags, and risk category.
FR-08 Answer copilot questions using structured risk context. Copilot cites calculated metrics and avoids advice wording.
FR-09 Generate a presentation-ready risk memo. Memo includes total risk, drivers, stress losses, commodity exposure, news flags, and caveats.

5. Risk Engine Specification

Return calculation

  • Use adjusted close where available; otherwise document close-price limitation.
  • Compute simple returns for interpretability and optionally log returns for robustness checks.
  • Align all return series by trading date and report dropped/missing observations.

Portfolio aggregation

  • Weight each holding by current market value divided by modeled portfolio value.
  • Daily portfolio return equals weighted sum of holding returns.
  • Unsupported assets are excluded from modeled risk and shown separately.
Metric Method Output
Historical VaR Empirical quantile of portfolio return distribution at 95% and 99% confidence. 1-day and square-root-time 10-day VaR amount and percentage.
Expected Shortfall Average of returns worse than VaR threshold. Tail loss amount and percentage.
Rolling volatility Rolling standard deviation annualized with trading-day factor. Trend chart and latest volatility value.
Drawdown Cumulative portfolio value path versus running peak. Maximum drawdown and current drawdown.
Component VaR Approximate contribution from each holding using covariance and portfolio weights. Top risk contributors by amount and percentage contribution.
Stress testing Apply scenario shock vector to holdings/sectors/factors. Scenario loss, affected sectors, top loss contributors.

6. Commodity Spillover Specification

The commodity module is a risk-driver layer, not a replacement for total portfolio risk. It answers whether commodity shocks are historically associated with portfolio or sector movements after short lags.

Step Prototype method Production extension
Factor construction Daily returns and volatility z-scores for crude, Brent, gold, copper, natural gas. Licensed/validated commodity feed, currency adjustment, Indian market calendar alignment.
Lag scan Test lags 1, 2, 3, 5, and 10 trading days against sector/portfolio returns. Rolling window lag stability, regime detection, confidence bands.
Regression Portfolio/sector return as function of lagged commodity returns and volatility shocks. Regularized multi-factor model and residual diagnostics.
Risk score Combine absolute beta, significance, sector exposure, and recent shock magnitude. Backtested early-warning score with alert thresholds. 
commodity_sensitivity_score =
  sector_weight
  * normalized_abs_lag_beta
  * recent_factor_shock_zscore
  * confidence_multiplier

7. News Risk Specification

MVP rule Store and display headline metadata only: headline, link, source, timestamp, matched sector/ticker, and risk tags. Do not scrape full article bodies in V1.
Risk tag Keyword examples Possible portfolio implication
Commodity crude, Brent, oil, gold, copper, gas, supply Margin pressure, input cost change, sector volatility.
Regulatory SEBI, RBI, circular, penalty, disclosure, compliance Event risk for financials, brokers, listed companies.
Macro inflation, repo, rupee, rates, GDP, fiscal Broad market and interest-sensitive sector risk.
Earnings results, profit, margin, revenue, guidance Single-stock volatility and gap risk.

8. Dashboard Specification

Overview

  • Total portfolio value.
  • Modeled value percentage.
  • 99% 10-day VaR and ES.
  • Data freshness indicator.

Risk drivers

  • Top component VaR holdings.
  • Sector concentration.
  • Correlation heatmap.
  • Largest drawdown periods.

Context

  • Commodity sensitivity cards.
  • Stress scenario comparison.
  • RSS risk flags.
  • Copilot Q&A panel.
State UI behavior
No portfolio loaded Show three clear actions: connect Zerodha, upload CSV, load sample portfolio.
Partial data Show risk metrics for modeled holdings and a warning for excluded instruments.
Stale market data Display last-updated timestamp and continue using cached data with warning.
Failed auth/API Show human-readable error and fallback actions without exposing secrets.

9. Non-Functional Requirements

Category Requirement
Performance Sample portfolio dashboard should render in under 60 seconds on a normal laptop.
Reliability Demo must work with no live Zerodha account and no external network after sample data is prepared.
Privacy Prototype should not persist access tokens or upload raw holdings to a hosted service by default.
Auditability Every risk output should include calculation method, input date range, and data quality status.
Safety Copilot must refuse buy/sell/hold recommendations and redirect to risk explanation.

Phase 5 Production-readiness plan. Hosting, security, token storage, compliance, and cost changes documented.

11. Acceptance Test Scenarios

  • CSV import loads a portfolio and produces a modeled/unsupported split.
  • Sample portfolio produces 95% and 99% VaR plus ES without internet access.
  • Missing price data produces a warning, not a crash.
  • Commodity module reports "not significant" when lag relationships are weak.
  • RSS failure displays stale/unavailable news state without affecting core risk calculations.
  • Copilot answers "What is my biggest risk?" using top component VaR and concentration outputs.
  • Copilot refuses "Should I buy more of stock X?" and offers risk context instead.
